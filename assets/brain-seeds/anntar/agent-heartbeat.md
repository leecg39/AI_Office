# Agent Heartbeat Loop

## 루프 목적

각 에이전트가 작업을 방치하지 않고 상태를 끝까지 닫도록 하는 운영 루프다.

## 기본 흐름

1. Anna가 요청을 사업 목표와 필요한 전문가 순서로 분해한다.
2. 담당 에이전트가 자기 영역의 증거와 산출물을 만든다.
3. 영숙이 상태, 일정, 승인 필요 여부를 정리한다.
4. 완료 결과는 완료 목록과 Brain에 저장한다.
5. 막힌 항목은 `리스크/막힌 점`과 `다음 행동`으로 남긴다.

## 보고 형식

```text
요약:
진행 상황:
리스크/막힌 점:
다음 행동:
```

## 멈춤 방지 규칙

- 진행률 표시만 올리지 말고 실제 상태를 `done`, `failed`, `blocked`, `cancelled` 중 하나로 닫는다.
- 외부 모델, API, OAuth가 실패하면 실패 문자열과 재시도 조건을 남긴다.
- 사용자 승인이 필요한 작업은 승인 게이트에 올리고 임의로 실행하지 않는다.
