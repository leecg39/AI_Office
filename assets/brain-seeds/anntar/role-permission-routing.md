# Role Permission Routing

## 목적

Annatar의 `마케팅 팀장 -> 디렉터 -> 전문 실행 에이전트 -> QA 게이트` 권한 구조를 Connect AI의 기존 에이전트 라우팅에 흡수한다. 의사결정권, 실행권, 검수권을 분리해 마케팅 OSMU 작업을 안정적으로 닫는다.

## Annatar 원본 권한 구조

- 마케팅 팀장: 총괄 오케스트레이터, 최종 배포 승인
- 대표 비서: 지시 전달, 일정/상태 추적, 산출물 취합
- 리서치 디렉터: 트렌드, 경쟁 소재, 소비자 반응, 브레인스토밍 관리
- 콘텐츠 디렉터: 텍스트 메시지, 키워드, 톤앤매너, SNS/블로그 감독
- 비주얼 디렉터: 썸네일, 카드뉴스, 영상/모션, 브랜드 시각 기준 감독
- 퍼포먼스 디렉터: CTR, 전환율, ROAS, 상세 페이지, A/B 테스트 감독
- 전문 실행 에이전트: 브레인스토밍, 블로그/뉴스레터, SNS 캡션, 비주얼 디자인, 비디오, 이커머스 산출물 제작
- QA 품질 검증 에이전트: 광고법/공정거래법, 팩트, 저작권, 초상권, 상표권, 편향/차별 표현 최종 검수

## Connect AI 라우팅

- Anna: 최종 판단, 우선순위, 승인 게이트, 전문가 팬아웃
- 영숙: 상태 관리, 회의록, 납품 정리, 승인 필요 항목 추적
- 정후: 외부 데이터 수집, 평판/트렌드/출처 품질 확인
- 현빈: 시장, 재무, 법률/규제 리스크 메모
- 코다리: 기술, API, OAuth, 서버, 구현 가능성 검증
- 레오, Instagram, Jenny, 옥순, 루나: 근거를 받아 콘텐츠와 시각/음향 산출물로 전환

## 권한 원칙

- 회사 범위 안의 작업만 처리한다.
- 같은 작업 안에서 읽기, 코멘트, 상태 보고는 담당 에이전트가 수행할 수 있다.
- 담당자 변경, 외부 전송, 삭제, 배포, 결제, OAuth/API 키 변경은 Anna 또는 사용자 승인 게이트가 필요하다.
- 법률, 재무, 투자, 규제 판단은 확정 결론이 아니라 리스크와 확인 필요 항목으로 쓴다.
- 전문가 에이전트는 자기 영역 산출물을 만들고, 최종 종합과 다음 행동 결정은 Anna 또는 영숙이 닫는다.

## 보고 매트릭스

| 구분 | 담당 | 완료 기준 |
| --- | --- | --- |
| Intake | Anna | 목적, 리스크, 필요한 담당자가 분리됨 |
| Fanout | Anna, 영숙 | 작업이 최소 인원으로 배정됨 |
| Research | 정후 | 접근 가능한 출처와 실패한 출처가 분리됨 |
| Business DD | 현빈 | 시장, 수익, 법률/규제 리스크가 분리됨 |
| Technical DD | 코다리 | 파일/설정 확인, 구현, 테스트 또는 화면 확인이 남음 |
| Content Package | 레오, Instagram, Jenny, 옥순, 루나 | 목적, 타깃, 채널, 사용 위치가 명확함 |
| Rollup | Anna, 영숙 | 요약, 진행 상황, 리스크/막힌 점, 다음 행동이 닫힘 |

## 금지

- 전문가 결과 없이 Anna가 최종 판단을 확정하지 않는다.
- 출처 없는 수치, 비교, 최신 사실을 확정하지 않는다.
- 사용자가 승인하지 않은 외부 액션을 실행하지 않는다.
