# Hermes Bootstrap Playbook

## 목적

Paperclip Company Factory의 Hermes-first 흐름을 Connect AI의 안전한 부트스트랩/설정 변경 방식으로 변환한다. 자연어 요청으로 회사, 워크플로, 대규모 설정을 만들 때 사용한다.

## 기본 흐름

1. 사용자가 만들려는 회사, 워크플로, 기능을 한 문장으로 설명한다.
2. Anna가 dry-run 계획을 만든다.
3. 계획에는 생성될 항목, 수정될 파일, 위험 액션, 필요한 승인, 검증 방법을 포함한다.
4. 사용자가 승인하면 실제 생성 또는 변경을 실행한다.
5. 실행 후에는 생성된 ID, 파일, URL, 서버 상태, 다음에 해볼 명령을 보고한다.

## Connect AI 적용 규칙

- 위험 액션은 dry-run 먼저 보여준다.
- 실제 bootstrap, 대량 파일 생성, 배포, OAuth/API 설정 변경은 승인 후 실행한다.
- raw JSON만 던지지 말고 사람이 읽을 수 있는 한국어 보고서를 함께 제공한다.
- 서버가 멈추면 포트, PID, health endpoint, 로그 경로 순서로 복구한다.
- 생성 결과는 Brain에 저장 가능한 Markdown 형태로 남긴다.

## Dry-run 보고서 형식

```text
요약:
생성/수정 예정:
승인 필요:
검증 방법:
리스크/막힌 점:
다음 행동:
```

## 완료 보고서 형식

```text
요약:
생성/수정한 항목:
검증:
실행 중인 서버/URL:
다음 행동:
```

## 금지

- 승인 없이 실제 결제, 배포, 외부 전송을 하지 않는다.
- 비밀키, OAuth 토큰, 개인 정보를 보고서나 Brain에 저장하지 않는다.
- 실패한 설치나 서버 상태를 성공처럼 보고하지 않는다.
