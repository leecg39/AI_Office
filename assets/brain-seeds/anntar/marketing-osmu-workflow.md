# Marketing OSMU Workflow

## 목적

Annatar 마케팅 팀의 OSMU 제작 흐름을 Connect AI 콘텐츠 에이전트의 표준 작업 방식으로 이식한다. 영상, 전사, 고객 브리프 1개를 블로그, 뉴스레터, SNS, 썸네일, 숏폼, 광고 소재, 성과 추적 계획으로 변환할 때 사용한다.

## 표준 입력

```json
{
  "source_video_or_transcript": "",
  "customer_brand": "",
  "target_audience": "",
  "offer": "",
  "channels": [],
  "primary_cta": "",
  "legal_constraints": "",
  "brand_assets": ""
}
```

필수 입력은 원본 자료, 브랜드, 타깃 고객, 제안, CTA다. 비어 있는 선택 입력은 `가정` 또는 `고객 확인 필요`로 표시한다.

## 표준 출력

```json
{
  "content_package": {},
  "visual_package": {},
  "video_package": {},
  "ad_creative_package": {},
  "tracking_plan": {},
  "qa_report": {}
}
```

## 작업 순서

1. Anna 또는 영숙이 고객 브리프를 표준 입력으로 정리한다.
2. 정후가 고객 언어, 경쟁 소재, 구매 동기, 금지 표현을 조사한다.
3. 현빈이 제안, CTA, 수익/성과 지표, 규제 리스크를 확인한다.
4. 레오와 Instagram이 채널별 콘텐츠 각도와 업로드 가설을 만든다.
5. Jenny가 블로그, 뉴스레터, SNS, 광고 카피 초안을 만든다.
6. 옥순이 썸네일, 카드뉴스, 광고 이미지 브리프를 만든다.
7. 루나가 숏폼/영상용 사운드와 자막/타이틀 요구사항을 정리한다.
8. 영숙이 QA 결과와 고객 납품 패키지를 정리한다.

## QA 체크리스트

- 모든 산출물이 같은 브랜드 보이스와 CTA를 따른다.
- 채널별 형식, 길이, 해시태그, 썸네일/이미지 요구사항을 지킨다.
- 출처 없는 숫자, 비교, 성과 주장은 단정하지 않는다.
- 영상, 음악, 이미지, 얼굴, 로고, 폰트의 사용권을 확인한다.
- 의료, 금융, 법률, 투자, 안전 관련 문구는 고객 확인 필요 항목으로 표시한다.
- 최종 판정은 `통과`, `수정 필요`, `반려` 중 하나로 닫는다.

## 상품 패키지 기준

- Starter: 원본 1개를 블로그, 뉴스레터, SNS, 썸네일 브리프, 카드뉴스 구성안, QA 리포트로 변환
- Growth: Starter에 숏폼 스크립트, 광고 카피, 랜딩/상세페이지 초안, 콘텐츠 캘린더를 추가
- Performance: Growth에 UTM/이벤트 추적, A/B 테스트 가설, 주간 성과 리포트 템플릿을 추가
