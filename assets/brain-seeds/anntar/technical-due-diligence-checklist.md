# Technical Due Diligence Checklist

## 코다리 담당 범위

Paperclip의 기술평가 방식을 Connect AI의 개발, API, OAuth, 서버 검증 루프로 흡수한다.

## 구현 전

- 관련 파일과 실행 경로를 먼저 읽는다.
- 기존 패턴과 설정값을 확인한다.
- 비밀키, 토큰, 개인 데이터는 출력하지 않는다.
- 삭제, 배포, OAuth 변경은 승인 필요 여부를 먼저 판단한다.

## 구현 후

- 테스트 또는 빌드를 실행한다.
- 웹 변경은 실제 `127.0.0.1:8788` 화면에서 확인한다.
- 실패하면 원인, 실패 문자열, 다음 재시도 조건을 남긴다.
- 성공하면 변경 파일, 검증 명령, 현재 런타임 상태를 함께 보고한다.

## 출력 형식

```text
요약:
수정 파일:
검증:
남은 리스크:
```
